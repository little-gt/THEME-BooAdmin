# 🔐 安全策略 (Security Policy)

感谢您对 **BooAdmin** 的安全关注！我们致力于为 Typecho 用户提供一个既美观又安全的后台环境。我们非常欢迎来自社区的负责任的安全漏洞披露。

---

## 📌 支持的版本

为了确保所有用户都能获得最新的安全修复，我们仅为最新版本提供安全更新。请始终确保您使用的是最新版本的 BooAdmin。

| 版本 | 状态 |
| :--- | :--- |
| **1.0.0** | ✅ 最新版本 |

---

## 📮 报告安全漏洞

如果您发现了任何潜在的安全漏洞，我们恳请您遵循负责任的披露原则，**不要在公开场合（如 GitHub Issues 或论坛）中发布细节**。这有助于防止漏洞在修复前被恶意利用。

### 📧 安全联系邮箱

[coolerxde@gt.ac.cn](mailto:coolerxde@gt.ac.cn)

为了让我们能快速识别和处理您的报告，请在邮件标题中包含以下格式：
`[Security] BooAdmin - <漏洞简述>`

### 🔐 需要加密通信？

如果您希望使用加密方式进行沟通，请在初次联系的邮件中告知我们。我们将提供 **PGP 公钥**，以便您能够安全地发送漏洞的详细信息和 PoC。

---

## 🧾 报告中建议包含的信息

一份详细的报告能够极大地帮助我们快速定位、复现并修复问题。我们建议您的报告中包含以下内容：

*   **影响版本**：例如 `v1.0.0` 或特定的 commit hash。
*   **环境信息**：Typecho 版本、PHP 版本、服务器软件 (Nginx/Apache)、浏览器及操作系统。
*   **漏洞类型**：如跨站脚本 (XSS)、跨站请求伪造 (CSRF)、信息泄露、权限绕过等。
*   **复现步骤**：提供清晰、可操作的步骤，让我们能够重现该漏洞。
*   **PoC (Proof of Concept)**：提供最小化的复现代码或请求示例（强烈推荐）。
*   **影响评估**：您认为该漏洞的潜在风险级别（高/中/低）。
*   **修复建议** (可选)：如果您有修复方案或补丁，我们将不胜感激。

---

## 📢 披露政策

我们遵循 **负责任披露 (Responsible Disclosure)** 的原则：

*   我们承诺在收到报告后 **48 小时内** 给予初步确认。
*   我们将与您保持沟通，告知修复进度。
*   在发布修复补丁或新版本之前，请不要公开漏洞的任何细节。
*   修复完成后，我们将在更新日志或安全公告中对您进行致谢。如果您希望匿名或使用特定署名，请在邮件中告知。

---

## 📦 安全范围

本安全策略主要覆盖由 **BooAdmin** 主题自身引入的代码：

*   ✅ `admin` 目录下的所有 PHP 模板文件。
*   ✅ `admin/js` 目录下的自定义 JavaScript 文件。
*   ✅ `admin/css` 目录下的自定义 CSS 文件。

以下情况**不属于**本主题的安全范围：

*   ❌ Typecho 核心自身的漏洞。
*   ❌ 第三方 Typecho 插件引入的漏洞。
*   ❌ 服务器配置不当导致的安全问题（如文件权限、PHP 配置等）。
*   ❌ 社交工程或密码破解等非代码层面的攻击。

如果发现漏洞是由第三方插件引起的，我们将尽力通知插件作者。
