# 🔐 安全策略 (Security Policy)

感谢您对 **BooAdmin** 的关注！我们致力于为 Typecho 用户提供既美观又安全的后台环境。虽然这是一个由社区维护的开源项目，但我们非常重视每一个安全反馈。

---

## 📌 支持的版本

为了确保资源能集中在最新的代码库上，我们目前仅对以下版本提供安全支持：

| 版本 | 状态 | 说明 |
| :--- | :--- | :--- |
| **1.0.1** | ✅ **最新正式版** | 推荐使用。提供全面的安全修复和功能维护。 |
| **1.0.0** | ⚠️ **维护模式** | 仅修复严重的、高危的安全漏洞。 |

---

## 📮 报告安全漏洞

如果您在 BooAdmin 中发现了潜在的安全漏洞，请**不要**在 GitHub Issues 或其他公共论坛中直接披露。这有助于防止漏洞在修复发布前被恶意利用。

请遵循以下步骤进行**负责任的披露**：

### 1. 联系方式
请将漏洞详细信息发送至安全联系邮箱：
📧 **[coolerxde@gt.ac.cn](mailto:coolerxde@gt.ac.cn)**

### 2. 邮件格式
为了帮助我们要快速分类和处理，请在邮件标题中使用以下格式：
`[Security] BooAdmin - <漏洞简述>`

### 3. 报告内容
为了加快修复速度，请尽量包含以下信息：
*   **影响版本**：例如 v1.0.1。
*   **漏洞类型**：如 XSS, CSRF,越权访问等。
*   **复现步骤**：清晰的文字描述或复现视频。
*   **PoC (概念验证)**：如果可能，提供一段最小化的复现代码或请求包。

---

## 📢 响应与披露流程

1.  **确认收到**：我们承诺在收到报告后的 **48 小时内** 给予回复确认。
2.  **评估修复**：我们将评估漏洞的严重程度，并着手开发修复补丁。在此期间，请您保持静默。
3.  **发布更新**：修复完成后，我们将发布新版本（如 v1.0.2），并在更新日志中注明。
4.  **公开致谢**：在漏洞修复并发布后，如果你愿意，我们将在安全公告或 Release Note 中对您的贡献进行致谢。

---

## 📦 安全范围

本策略仅涵盖 **BooAdmin 主题本身** 的代码：

*   ✅ `admin/` 目录下由本主题修改或新增的 PHP 文件。
*   ✅ `admin/js/` 和 `admin/css/` 目录下的主题资源文件。

**以下情况不在本项目的安全处理范围内：**

*   ❌ **Typecho 核心** 自身的漏洞（请直接向 Typecho 官方报告）。
*   ❌ **第三方插件** 引入的安全问题。
*   ❌ 服务器环境配置不当（如 PHP 版本过低、目录权限设置错误等）。
*   ❌ 针对管理员账号的暴力破解或社交工程攻击。
