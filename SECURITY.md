## 📌 支持的版本

为了将精力集中在提供更优质的代码上，我们目前仅对以下版本提供安全更新：

| 版本分支 | 维护状态 | 策略说明 |
| :--- | :---: | :--- |
| **v1.1.x** (如 1.1.3+) | ✅ **全面支持** | **当前主推正式版**（基于 TailwindCSS 重构）。提供最高优先级的安全更新与漏洞修复。 |
| **v1.0.x** (如 1.0.2) | ⚠️ **仅限安全** | 旧版 Bootstrap 分支。仅对**严重高危漏洞**提供修复，支持至 v1.1.1 发布后的 60 天。建议尽快升级。 |
| **< 1.0.2** | ❌ **停止支持** | 已进入 EOL (生命周期结束)，不再提供任何形式的更新。 |

## 📮 如何报告安全漏洞？

> **⚠️ 核心原则**：如果您发现了潜在的安全漏洞，请**绝对不要**在 GitHub Issues 或其他公共论坛中直接披露，以免漏洞在修复前被恶意利用。请遵循**负责任的披露流程**。

### 1. 投递邮箱
请将漏洞的详细信息私密发送至我们的安全联系邮箱：
📬 **[coolerxde@gt.ac.cn](mailto:coolerxde@gt.ac.cn)**

### 2. 邮件标准格式
为了帮助我们快速分类、评估和响应，请在邮件标题中使用以下格式：
<kbd>[Security] BooAdmin - 漏洞简述 (如：XSS / CSRF)</kbd>

### 3. 建议包含的信息
为加快修复速度，请尽量在邮件中包含以下内容：
- **🎯 影响版本**：例如 v1.1.3，或说明是否影响所有版本。
- **🐛 漏洞类型**：如 XSS, CSRF, SQL 注入, 文件上传漏洞, 越权访问等。
- **💥 危害评估**：说明漏洞可能造成的实际危害（如数据泄露、提权、接管后台等）。
- **📝 复现步骤**：清晰的文字描述或演示视频，确保我们能够重现问题。
- **💻 PoC 代码**：如果可能，提供一段最小化的验证代码或 HTTP 请求包。
- **⚙️ 环境信息**：相关的 PHP 版本、Typecho 版本及浏览器类型。

## ⏱️ 响应与披露流程

1. **📨 确认接收**：我们承诺在收到报告后的 **48 小时内** 给予回复确认。
2. **🔍 评估修复**：我们将紧急评估漏洞严重程度，并着手开发补丁。**此期间请您保持静默，切勿公开。**
3. **🚀 发布更新**：修复完成后，我们将发布新版本（如 v1.1.x），并在更新日志中隐晦注明安全修复内容。
4. **🎉 公开致谢**：在补丁发布且多数用户更新后，如果您愿意，我们将在安全公告或 Release Note 中对您的贡献进行致谢（请在邮件中明确告知是否需要署名及您的 ID）。

## 🛡️ 后台安全最佳实践指南

为了最大化保障您的博客安全，除了使用安全的主题外，我们强烈建议您：
- **🔄 保持更新**：始终使用最新版本的 Typecho 和 BooAdmin 主题。
- **🔐 强密码与 HTTPS**：全站启用 SSL 证书（HTTPS 加密），并定期更换复杂的管理员密码。
- **💾 异地备份**：养成定期备份数据库和网站目录的习惯。
- **🚧 访问限制**：建议通过服务器配置（如 Nginx）或面板（如宝塔、云锁），限制 `/admin` 目录仅允许特定 IP 访问，或增加基础的 HTTP 身份验证。
- **🔌 审核插件**：仅从可信赖的来源安装插件，并及时清理不用的旧插件。
